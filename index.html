API 接入文档

📌 概述

数据集开放 API 允许持有有效 Key 的系统通过标准 HTTP 接口查询指定数据集的数据。
API 部署在内网，调用方须在内网或 VPN 环境下访问。

🌐 Base URL：https://data-api.987ht.net
🔒 仅限内网 / VPN 访问

🔑 鉴权方式

所有请求须在 Header 中携带 Authorization，格式为 Bearer Token：

Authorization: Bearer dk_live_a7f3c9e2b8d1047f3a9c2e8b1d047f3a
⚠️ Key 由管理员手动生成，请联系数据平台管理员申请。Key 泄露请立即联系管理员吊销。

📥 查询接口

GET /v1/dataset/{dataset_id}/query

请求参数

参数位置类型必填说明
dataset_idPathstring是数据集唯一标识，可在数据集列表查看
limitQueryinteger否返回行数，默认 100，不超过数据集配置的单次上限
offsetQueryinteger否分页偏移量，默认 0

完整请求示例

curl -X GET \
  "https://data-api.987ht.net/v1/dataset/ds_charge_order/query?limit=100&offset=0" \
  -H "Authorization: Bearer dk_live_a7f3c9e2b8d1047..." \
  -H "Content-Type: application/json"

返回格式

{
  "code": 0, // 0=成功，非0=失败
  "msg": "ok",
  "data": {
    "total": 7, // 本次返回行数
    "rows": [ // 数据行（对象数组）
      {"log_date": "2026-04-20", "total_amount": 128500.00},
      ...
    ],
    "cached": false, // 是否命中缓存
    "cached_at": null // 缓存生成时间（每日缓存模式有值）
  }
}

⚠️ 错误码说明

HTTP 状态码code含义处理建议
2000成功—
40110001未提供 Token 或 Token 格式错误检查 Authorization Header 格式
40110002Token 无效或已过期联系管理员重新生成 Key
40110003Token 已被吊销联系管理员
40310004该 Key 无权访问此数据集确认 dataset_id 与 Key 的授权数据集一致
42910005超出并发限制稍后重试，或联系管理员提升并发数
42910006超出今日调用次数上限等次日重置，或联系管理员提升日限额
50020001数据库查询异常重试，持续报错请联系数据平台

📋 限流规则

• 并发限制：每个数据集有最大并发请求数限制（由管理员配置），超出时返回 429（code 10005）
• 日调用限额：每个 Key 每天有调用次数上限（由管理员配置），次日 0:00 自动重置
• 重试建议：收到 429 后建议等待 1 秒后重试，避免突发大量重试加重压力

💾 缓存模式说明

数据集支持两种缓存模式：
• 实时查询（默认）：每次请求都执行 SQL，数据最新，但对数据库压力较大
• 每日缓存：每天凌晨 2:00 自动执行 SQL 并将结果缓存为文件，日间请求直接读缓存；返回值 cached: true，cached_at 为缓存生成时间
